# Worktree Session Setup

Налаштуй поточну worktree-сесію для ефективної роботи. Ти працюєш в **ізольованому git worktree** — окремій робочій директорії з власною гілкою.

## Крок 1: Назва сесії

Згенеруй коротку назву сесії з `$ARGUMENTS` (якщо передано) або з контексту розмови:
- Англійською, snake_case, 2-4 слова
- Наприклад: `gh_oauth_setup`, `fix_delete_cascade`, `refactor_middleware`

Використай `/rename <назва>` щоб встановити її. Це допоможе знайти сесію пізніше через `/resume`.

## Крок 2: Контекст роботи

Поясни користувачу ключові особливості worktree-сесії одним коротким абзацом:

- Файли тут **ізольовані** від інших worktree-сесій — зміни не впливають на інші гілки
- `.claude/skills/`, `.claude/agents/`, hooks з основного репо **доступні**
- При завершенні: `keep` зберігає worktree для продовження, `remove` прибирає гілку і файли

## Крок 3: Початок роботи

Запитай користувача що саме потрібно зробити і почни працювати над завданням.

---

## Поради для роботи в worktree

- Не коміть в main — коміт в гілку worktree, потім мерж через PR
- Якщо потрібен файл з іншої гілки — `git cherry-pick` або скопіюй вручну
- Для великих monorepos: `worktree.sparsePaths` в settings.json чекає лише потрібні директорії
- Субагенти можуть працювати в власних worktree через `isolation: "worktree"` в agent definition
