# Довідка форматів файлів

Шаблони для кожного файлу бази знань. Використовуйте при створенні або оновленні файлів.

> **ВАЖЛИВО:** Весь контент описів має бути українською. Технічні ідентифікатори (`--flag`, `VAR_NAME`, `/command`) залишаються як є.

---

## meta.md

```markdown
# Метадані бази знань

last_processed: X.Y.Z
last_updated: YYYY-MM-DD
total_entries: N

## Історія обробки

| Дата | Версія | Оновлені категорії | Записів додано |
|------|---------|--------------------|---------------|
| YYYY-MM-DD | X.Y.Z | cli, settings, core | 12 |
| YYYY-MM-DD | X.Y.Z-1 | mcp, hooks | 5 |
```

---

## index.md

```markdown
# Індекс бази знань Claude Code

> Останнє оновлення: YYYY-MM-DD | Остання версія: X.Y.Z
> Читайте файли категорій за потребою — НЕ завантажуйте всі одразу.

## Категорії

- [Основні можливості](categories/core.md) — Thinking, зусилля, план-режим, моделі, контекст
- [CLI](categories/cli.md) — Команди, прапорці, підкоманди
- [Налаштування](categories/settings.md) — Ключі settings.json, дозволи
...
```

---

## Шаблон категорійного файлу

```markdown
# [Назва категорії українською]

> Короткий опис що сюди входить

## [Підрозділ]

| Запис | Опис | Версія |
|-------|------|--------|
| `identifier` | Опис українською мовою | X.Y.Z |
| ~~`old-thing`~~ | [ЗАСТАРІЛО] причина | X.Y.Z |
```

Для простих списків (напр. keybindings):

```markdown
## Навігація

- `Ctrl+L` — Очистити екран + повний перемальовування (2.1.83)
```

## Шаблон реліз-снапшоту

```markdown
# Реліз X.Y.Z — YYYY-MM-DD

## Нові можливості
- Опис можливості (категорія: core)
- Опис можливості (категорія: cli)

## Виправлення
- Опис виправлення (категорія: windows)

## Важливі зміни
- Опис зміни (категорія: settings)
```
